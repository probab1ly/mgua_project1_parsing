# Безопасный запуск AI Regulation Monitor

## Запуск

```powershell
python run_server.py
```

Приложение будет доступно по адресу `http://127.0.0.1:5000`.
Для публикации в сети требуется HTTPS и обратный прокси. В таком случае задайте:

```powershell
$env:AI_MONITOR_HTTPS = "1"
```

## Учетные данные

Рабочие логины находятся в `users.json` в виде PBKDF2-хешей. Пароли в открытом
виде хранятся только в локальном файле `private_credentials.txt`, который
исключен из Git. Передавайте этот файл только по защищенному каналу.

Секрет Flask автоматически создается в `.ai_monitor_secret`. Файл также
исключен из Git. Для сервера лучше задать отдельный секрет длиной от 32 знаков:

```powershell
$env:AI_MONITOR_SECRET_KEY = "случайная-строка-длиной-не-менее-32-знаков"
```

## Gmail SMTP

Обычный пароль Google не подходит. Нужен отдельный пароль приложения,
созданный после включения двухэтапной аутентификации.

Используйте `email_environment.example.ps1` как образец и задайте пароль только
через переменную окружения:

```powershell
$env:AI_MONITOR_SMTP_PASSWORD = "пароль-приложения-без-пробелов"
```

Пароль не следует записывать в `email_config.json`. Если пароль когда-либо
попадал в Git или пересылался открытым текстом, его нужно отозвать в аккаунте
Google и создать новый.

## Проверка

```powershell
python -m unittest discover -s quality_assurance/tests -v
```

Контрольные проверки покрывают авторизацию, CSRF, ограничение перебора пароля,
заголовки безопасности, API-валидацию, открытые перенаправления, XSS-ссылки,
формулы Excel, конкурентную запись истории и повторную NLP-классификацию кэша.

## Ограничения внешних источников

Официальные сайты могут возвращать HTTP 403, менять HTML, блокировать
автоматические запросы или не отвечать вовремя. Парсер изолирует такие ошибки,
ограничивает длительность обновления и показывает состояние каждого источника.
Для заранее проверенных правовых страниц может использоваться резервная запись
из конфигурации, но неподтвержденный произвольный контент таким способом не
добавляется.

Перезапись истории Git для удаления старых секретов намеренно не выполнялась:
это изменяет идентификаторы коммитов и требует отдельного согласования со всеми
участниками репозитория.
